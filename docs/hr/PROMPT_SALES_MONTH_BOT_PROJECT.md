# Промпт для проекта: Telegram-бот учёта «Продажный месяц DK»

**Назначение:** использовать этот промпт при создании нового проекта, чтобы развернуть систему учёта баллов/победителей для программы «Продажный месяц» в компании DimKava.

---

## Контекст

Компания DimKava — retail кофе и кофемашин (Blasercafe, DeLonghi) в Грузии. 4 точки продаж в молах, у каждой — отдельный Telegram-чат сотрудников.

**Программа «Продажный месяц»:** 4 недели, каждая с своим фокусом (контакт, cross sale, карты + бонусы, соревнование точек). Цель — повысить средний чек, cross sale, подключение бонусных карт, применить регламенты через игру.

---

## Идея системы учёта

Сотрудники отправляют в чат своей точки **фото чека + текст с ключевым словом** (например, `CROSSALE`).  
Бот, добавленный в каждый чат, реагирует на такие сообщения: цитирует их и записывает в Google Таблицу **имя пользователя** и **количество сообщений** с этим ключевым словом.  
В конце недели по таблице определяют победителя.

---

## Что нужно реализовать

1. **Telegram-бот**
   - Работает в группах (чаты точек продаж).
   - Читает сообщения, где есть:
     - фото (или документ с изображением чека);
     - текст, содержащий ключевое слово недели.
   - При совпадении условий: отвечает цитатой (подтверждение, что сообщение засчитано) и сохраняет запись в Google Sheets.

2. **Google Таблица**
   - Структура: дата, имя пользователя, ID пользователя Telegram, ключевое слово, чат/точка.
   - Лист для агрегации: количество сообщений по пользователю за период (день/неделя).

3. **Ключевые слова по неделям**
   - Неделя 1: `КОНТАКТ` или `ЭНЕРГИЯ`
   - Неделя 2: `CROSSALE`
   - Неделя 3: `КАРТА` или `БОНУСЫ`
   - Неделя 4: по решению (например, `ТОЧКА` или оставить несколько слов).
   - Возможность задавать/менять ключевое слово через конфиг или команду.

4. **Ограничения (защита от спама)**
   - Не засчитывать повторные сообщения от одного пользователя в течение N минут (например, 5–10).
   - Опционально: лимит сообщений в день на пользователя (например, 20–30).

5. **Еженедельный отчёт**
   - Скрипт или функция: выборка из таблицы за неделю, подсчёт по пользователям, топ-3 или топ-N.
   - Вывод: в чат админа/общий чат или в отдельный лист таблицы.

## Технические требования

- Язык: Python (рекомендуется: `python-telegram-bot` или `aiogram`).
- Google Sheets API для записи и чтения.
- Хранение конфигурации: `.env` (токен бота, ID таблицы, ключевые слова).
- Деплой: возможность запуска на VPS, в облаке или локально (по решению команды).

## Критерии успеха

- Бот добавлен во все 4 чата точек.
- Сообщения с фото + ключевым словом корректно засчитываются и записываются.
- Таблица обновляется в реальном времени.
- В конце недели можно получить рейтинг по пользователям за 1–2 клика.
- Защита от спама/накрутки работает.
