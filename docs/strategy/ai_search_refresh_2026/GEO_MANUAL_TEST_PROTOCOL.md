# Протокол ручного GEO‑тестирования (без платных инструментов)

Цель: регулярно проверять, **появляется ли DimKava** в ответах AI и **правильно ли** описывается.

## 1) Движки для проверки (минимум)
- Google (Gemini / AI Overviews где доступно)
- Perplexity (удобно для цитат/источников)
- ChatGPT (для общей видимости и формулировок)

## 2) Частота
- **Раз в 2 недели** первые 2 месяца (когда быстро меняется сайт/GMB/FAQ)
- Затем **раз в месяц** стабильно

## 3) Набор запросов
Использовать матрицу из `QUERY_MATRIX.md`:
- 10 запросов EN
- 10 запросов KA
- 5 запросов RU (минимум)

Всегда включать «локальные» вариации:
- Tbilisi / Batumi / Georgia
- coffee beans / coffee shop / Delonghi service / coffee machine repair

## 4) Что фиксировать (в таблицу)
Минимальные колонки:
- Дата
- Движок (Gemini/Perplexity/ChatGPT)
- Язык (EN/KA/RU)
- Запрос
- **Упоминание DimKava?** (Да/Нет)
- **Ссылка/цитата на dimkava.ge или GMB?** (Да/Нет/Какая)
- **Точность** (0–2): 0 = ошибки, 1 = частично, 2 = точно
- **Сообщение бренда** (0–2): 0 = нет «полного цикла/наслаждения», 1 = частично, 2 = есть
- Комментарий: что исправить (страница/FAQ/GMB/внешнее упоминание)

## 5) Как оценивать «узнаваемость»
Сводные метрики за период:
- Presence rate = упоминаний / всех запросов
- Citation rate = цитат/ссылок / всех запросов
- Accuracy = средняя точность

## 6) Что делать по результатам
Если DimKava не упоминают:
- усилить «точки истины» (страницы локаций, сервис Delonghi, FAQ) и их структуру
- добавить ясные блоки «Key facts» в начале страниц
- синхронизировать GMB ↔ сайт ↔ контакты

Если упоминают, но с ошибками:
- исправить источник (обычно: GMB, сайт, FAQ)
- добавить на сайте явные факты в одном месте (и дату обновления)
