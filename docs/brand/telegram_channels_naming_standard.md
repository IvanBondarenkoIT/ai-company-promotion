### Проблемы текущей системы:

❌ **Отсутствие единого стандарта** — разные форматы, языки, стили  
❌ **Смешение языков** в одном названии (русский + грузинский + английский)  
❌ **Неинформативность** — некоторые названия не отражают назначение  
❌ **Отсутствие брендинга** — не везде присутствует "DimKava" или "DK"  
❌ **Длинные названия** — сложно читать и запоминать  
❌ **Нет категоризации** — непонятна структура каналов

## 🎯 ПРИНЦИПЫ ИМЕНОВАНИЯ

### 1. **КРАТКОСТЬ И ЯСНОСТЬ**
- Максимум 3-4 слова
- Понятно с первого взгляда
- Легко запомнить и найти

### 2. **ЕДИНООБРАЗИЕ**
- Единый формат для всех каналов
- **Отображаемое имя** — на английском (см. формат ниже); без дублирования каналов по языкам
- Единая структура

### 3. **БРЕНДИНГ**
- Присутствие "DimKava" или "DK" в названии
- Соответствие философии компании
- Профессиональный вид

### 4. **ИНФОРМАТИВНОСТЬ**
- Название отражает назначение канала
- Понятно, для кого канал
- Понятно, какой контент внутри

### 5. **МАСШТАБИРУЕМОСТЬ**
- Легко добавить новые каналы
- Логичная структура для расширения
- Гибкость для будущего роста

### 6. **МНОГОЯЗЫЧНОСТЬ**
- **Английский** — для **отображаемого имени** канала в Telegram (список чатов, поиск)
- **Один логический канал** на тему; отдельные каналы под RU/KA **не создаются**
- Переписка и материалы внутри — на RU/EN/KA; при необходимости **поле About и закреп** дублируют суть на нескольких языках
- Подробные тексты для сотрудников — в [telegram_channels_recommendations.md](telegram_channels_recommendations.md)

## 📐 СТРУКТУРА СТАНДАРТА

### Формат названия:

```
[Префикс] [Категория] [| Детали]
```

**Где:**
- **Префикс**: `DK` или `DimKava` (для публичных каналов)
- **Категория**: тип канала на английском (Sales, Training, Regulations, и т.д.)
- **Детали** (после `|`): локация, тема или уточнение — **тоже на английском**
- **Имя в списке чатов**: только латиница в выбранном формате; **без** суффиксов `| RU` / `| KA` в названии
- **Общение внутри**: все языки (RU/EN/KA)

### Категории каналов:

1. **ВНУТРЕННИЕ ОПЕРАЦИИ** (Internal Operations)
   - Склад и логистика
   - Инвентаризация
   - Продажи и отчетность

2. **РЕГЛАМЕНТЫ И СТАНДАРТЫ** (Regulations & Standards)
   - Правила работы
   - Стандарты обслуживания
   - Процедуры

3. **ОБУЧЕНИЕ И РАЗВИТИЕ** (Training & Development)
   - Обучение сотрудников
   - Мастер-классы
   - Развитие навыков

4. **КОНТЕНТ И МАРКЕТИНГ** (Content & Marketing)
   - Контент для соцсетей
   - Фото и видео
   - Публичные каналы

5. **ОПЕРАЦИОННЫЕ ЗАДАЧИ** (Operational Tasks)
   - Чистка оборудования
   - Ежедневные задачи
   - Контроль качества

## 🚀 РУКОВОДСТВО ДЛЯ БУДУЩИХ КАНАЛОВ

### Шаблон для создания нового канала:

1. **Определить категорию:**
   - Внутренние операции
   - Регламенты и стандарты
   - Обучение и развитие
   - Контент и маркетинг
   - Операционные задачи

2. **Язык переписки и материалов:** любой (RU/EN/KA); **имя канала** — только в формате ниже на английском.

3. **Применить формат:**
   ```
   DK [Категория] | [Детали]
   ```

4. **Примеры для будущих каналов:**

   **Публичные каналы (для клиентов):**
   - `DimKava | News`
   
   **Внутренние каналы:**
   - `DK Reports | Daily`
   - `DK Reports | Weekly`
   - `DK Reports | Monthly`
   
   **Специализированные:**
   - `DK Master Classes | Barista`
   - `DK Service | Requests`
   - `DK Clients | Reviews`
   
   **Примечание:** Все названия на английском. Внутри каналов общение на всех языках (RU/EN/KA).

### Правила именования:

✅ **ПРАВИЛЬНО:**
- `DK Sales` — понятно, структурировано, на английском
- `DK Inventory | Batumi` — четкая структура, английский
- `DK Training` — единый канал для всех языков
- `DimKava | News` — публичный канал с полным названием

❌ **НЕПРАВИЛЬНО:**
- `склад заказы` — нет префикса, неструктурированно, не на английском
- `Система инвентаризации и учета Палиашвили` — слишком длинно, не на английском
- `Регламенты Дiм Кава` — смешение языков, не на английском
- `content` — слишком общее, нет брендинга, не на английском
- `DK Продажи` — на русском вместо английского

### Рекомендации по структуре:

1. **Используйте разделитель `|`** для категоризации
2. **Префикс `DK`** для внутренних каналов
3. **Префикс `DimKava`** для публичных каналов
4. **Не плодить каналы по языкам** — RU/KA выносим в About и закреп (см. [telegram_channels_recommendations.md](telegram_channels_recommendations.md))
5. **Короткие названия** — максимум 4 слова (в имени)
6. **Единый формат** — для всех каналов одной категории

## Миграция и технические идентификаторы

**Имя в списке чатов и @username — разные сущности.** Стандарт выше задаёт **отображаемое имя** (title). Публичная ссылка вида `https://t.me/...` зависит от **username** (латиница, уникальность). После смены username старые ссылки перестают работать — заранее обновить индекс-канал и регламенты.

| Действие | Примечание |
|----------|------------|
| **Матрица перехода** | Для каждого старого канала: старое имя → новое EN-имя → `t.me/...` → дата → ответственный. Шаблон: [telegram_channels_inventory.md](telegram_channels_inventory.md) |
| **Владельцы** | Назначить основного и запасного администратора (право менять имя и username в Telegram) |
| **Канал vs группа** | Канал — вещание; для обсуждений может быть **группа/супергруппа** — те же правила префикса `DK` в названии |
| **Форум с топиками** | Альтернатива четырём каналам инвентаризации — одна группа-форум с топиками по локациям; оценить по объёму сообщений |
| **Контент с лицами** | В `DK Content` / `DK Latte Art` — соблюдать внутренние правила согласия на съёмку |
| **Боты** | Имена ботов (например, учёт по точкам) логично согласовать с префиксом `DK` / `DimKava` — см. [PROMPT_SALES_MONTH_BOT_PROJECT.md](../hr/PROMPT_SALES_MONTH_BOT_PROJECT.md) |
| **Экспорт истории** | При слиянии каналов учитывать ограничения Telegram на историю; для критичных цепочек — заранее экспорт по политике компании |
